Decision: Use static API keys for initial auth implementation instead of Google OAuth.

Rationale:
- Zero external dependencies
- Faster development iteration
- OAuth can be added later as enhancement
- Doesn't change queue logic or architecture

Implementation approach:
- API keys stored in Worker environment variables
- Keys map to user emails for queue isolation
- Authorization: Bearer <key> header
- CLI stores key in config file

Google OAuth moved to future enhancement (issue #64)
