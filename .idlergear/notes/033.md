Design: Persistent Storage API for Forebay

## API Endpoints

### Document/KV Storage Operations

**PUT /store/{key}**
- Store or update a document
- Body: JSON object with `value` field
- Returns: {success, key, size, timestamp}

**GET /store/{key}**
- Retrieve a document
- Returns: {key, value, metadata: {created, updated, size}}
- 404 if not found

**DELETE /store/{key}**
- Delete a document
- Returns: {success, key, deleted_at}

**GET /store**
- List all stored documents (with prefix filter support)
- Query params: ?prefix=xxx&limit=100
- Returns: {documents: [{key, size, created, updated}]}

## Storage Structure

**KV Keys:**
- `store:{key}` - actual document data
- `store:index` - array of document keys (for listing)
- `store:meta:{key}` - metadata (created, updated, size)

**Document Format:**
```json
{
  "value": <any JSON>,
  "metadata": {
    "created_at": timestamp,
    "updated_at": timestamp,
    "size_bytes": number
  }
}
```

## Use Cases

1. **Task List:**
   - PUT /store/tasks/task-123
   - GET /store/tasks/task-123
   - GET /store?prefix=tasks/

2. **Notebooks:**
   - PUT /store/notes/2026-01-05-session
   - GET /store/notes/2026-01-05-session

3. **Configuration:**
   - PUT /store/config/app-settings
   - GET /store/config/app-settings

4. **Generic KV:**
   - PUT /store/user-preferences
   - GET /store/user-preferences
