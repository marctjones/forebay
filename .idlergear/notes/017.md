**Architecture Decision: Hybrid Approach**

**Technology Stack:**
- **Shared Library:** C# (.NET 9.0) with Forebay.Tasks
- **Terminal:** C# + Terminal.Gui (both CLI and interactive TUI)
- **Desktop:** C# + Avalonia (GNOME + Windows 11)
- **Storage:** SQLite for local persistence
- **Sync:** Forebay queues for cross-device sync

**Key Design Decisions:**

1. **C# + Terminal.Gui over Python + Textual**
   - Language consistency across entire project
   - Direct code reuse (no FFI needed)
   - Single binary distribution
   - Cross-platform terminal support

2. **Hybrid CLI + TUI Mode**
   - `forebay task` ‚Üí Interactive TUI
   - `forebay task add "..."` ‚Üí Scriptable CLI
   - Best of both worlds for different use cases

3. **Shared Core Library Pattern**

4. **Last-Write-Wins Sync Strategy**
   - Simple conflict resolution based on timestamps
   - Tombstone records for deletions
   - Offline-first with queue buffering
