Cloudflare Workers testing constraint: worker crate types (Request, Headers, etc.) use wasm-bindgen and cannot be instantiated in standard cargo test environment. Tests that try to create Request objects fail with "cannot call wasm-bindgen imported functions on non-wasm targets". Need to either:
1. Use wasm-pack test for WASM tests
2. Extract testable logic into pure functions that don't use worker types
3. Use integration tests that actually deploy to Cloudflare
Current approach: Extract business logic from worker types for unit testing
