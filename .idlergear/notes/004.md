**Phase 0 Research Findings: .NET and Python**

## .NET/C# on Cloudflare Workers

**Status:** Technically possible but **NOT RECOMMENDED** for production

**Approach:**
- Compile .NET 8 to WebAssembly via WASI SDK
- Deploy WASM to Cloudflare Workers

**Limitations:**
- Bundle size: 920KB gzipped (very tight on free tier 1MB limit)
- WASI support: Experimental in both .NET and Cloudflare
- Missing WASI functions: Cloudflare's impl lacks some .NET expects
- Practical use: Author states "impractical" for real apps on free tier
- Setup complexity: Very high

**Verdict:** Skip .NET for Phase 0. Not viable for production queue system.

## Python Workers - We're Using OUTDATED API!

**Problem:** Our test implementation uses old Pyodide/Transcrypt patterns:
- `from js import Response, Headers` (old way)
- `Response.new()` syntax (Transcrypt-era)
- Manual routing with url parsing

**Modern API (2024-2025):**
- Uses `from workers import WorkerEntrypoint, Response`
- Class-based with `fetch` method
- Memory snapshots for fast cold starts (2.4x faster than AWS Lambda)
- Full package support via Pyodide
- pywrangler CLI tool for modern workflow

**Performance Improvements:**
- **Cold starts:** 2.4x faster than AWS Lambda (with memory snapshots)
- **Packages:** Full Pyodide support (FastAPI, Langchain, httpx, Pydantic)
- **Best practices:** Use WorkerEntrypoint class pattern

**Our test used OLD API = unfair comparison!**
