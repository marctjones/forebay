**Language Decision Framework**

We need to choose TWO languages:
1. **Worker language** (runs on Cloudflare)
2. **Client language** (CLI, GUI, mobile)

**Worker Options:**
- TypeScript: ⭐⭐⭐⭐⭐ Native support, best docs
- Rust: ⭐⭐⭐⭐ WASM compile, great performance
- Python: ⭐⭐ Beta, Pyodide, limited ecosystem

**Client Options:**
- C#: Best for cross-platform GUI (Avalonia), learning goal
- Rust: Share code with Worker if both Rust, learning goal
- Python: User's strength, but GUI story weaker

**Key Insight:**
Worker and client CAN be different languages - they communicate via HTTP. The Worker is just a REST API endpoint.

**Phase 0 Goal:**
Test Worker options empirically before deciding. Measure:
- Bundle size
- Build time
- Cold start
- Request latency
- KV operations
- Developer experience

**Likely Outcomes:**
1. If Rust Worker performs well → Rust Worker + C# Client
2. If TypeScript is "good enough" → TypeScript Worker + C# Client
3. If Python viable → Python Worker + C# Client

Decision pending Phase 0 test results.
