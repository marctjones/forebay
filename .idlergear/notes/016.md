**TaskList App Architecture Decision: C# + Terminal.Gui over Python + Textual**

After analyzing options for the Ubuntu terminal TaskList app, decided on a **hybrid CLI + TUI approach using C# with Terminal.Gui**.

**Approaches Considered:**

1. **Simple CLI only** - Scriptable but not interactive
2. **Python + Textual TUI** - Rich UI but different language
3. **C# + Terminal.Gui TUI** - Rich UI with language consistency
4. **Hybrid (both CLI + TUI)** - Best of both worlds ✅

**Final Decision: C# + Terminal.Gui with Hybrid Mode**

**Rationale:**

✅ **Language Consistency** - Same C# as entire Forebay project
✅ **Code Reuse** - Directly use Forebay.Core library (no FFI)
✅ **Single Binary** - No Python runtime dependency
✅ **Cross-Platform** - Works on Linux, macOS, Windows
✅ **Microsoft-Backed** - Terminal.Gui actively maintained
✅ **Dual Mode** - Interactive TUI AND scriptable CLI commands

**Implementation:**

```bash
# Interactive TUI mode
forebay task

# Scriptable CLI mode  
forebay task add "Buy groceries"
forebay task list --json
forebay task complete <id>
```

**TUI Features:**
- Arrow key navigation
- Spacebar to toggle complete
- 'd' to delete
- Real-time sync indicator
- Status bar with keyboard hints
- Mouse support (optional)

**Architecture:**
```
Forebay.Cli → Forebay.Tasks → Forebay.Core
              (new shared lib)
```

**Why NOT Python + Textual:**
- Different language creates fragmentation
- Can't directly use Forebay.Core (would need FFI/subprocess)
- Requires Python runtime
- Separate binary distribution
- More complex maintenance

**Terminal.Gui Advantages:**
- Used by Microsoft projects (dotnet-monitor)
- Rich widget library
- Mouse + keyboard support
- Cross-platform terminal handling
- Natural extension of existing CLI
