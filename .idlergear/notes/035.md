Fixed storage API to support keys with slashes (hierarchical keys).

## Problem
Storage endpoints were returning 404 when keys contained slashes (e.g., `tasks/todo-list`, `config/app-settings`). The router was using `:key` parameter which only matches single path segments.

## Solution
Changed router from single-segment parameters to wildcard parameters:
- Changed `/store/:key` to `/store/*key` in all routes
- Updated storage handlers to trim leading slash from wildcard parameter
- Wildcard params include the leading `/`, so `trim_start_matches('/')` is used

## Files Changed
- worker/src/lib.rs - Updated routes to use `*key` wildcard
- worker/src/storage.rs - Updated all handlers to trim leading slash

## Testing
Confirmed working with hierarchical keys:
- `tasks/todo-list` ✅
- `config/app-settings` ✅  
- `notes/meeting-2026-01-05` ✅
- Prefix filtering working correctly

All 25 tests still passing, deployed to production successfully.
