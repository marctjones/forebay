Queue listing feature successfully implemented!

**Problem Solved:**
Cloudflare Workers KV doesn't support native key listing, so the list queues endpoint was returning empty arrays.

**Implementation:**
- Maintain queue index in KV at key "queues:index"
- Index stores sorted JSON array of queue names
- add_queue_to_index() - idempotent operation on push
- remove_queue_from_index() - removes from index on delete
- handle_list() - fetches index and gets stats for each queue

**Code Changes:**
- Added QUEUE_INDEX_KEY constant
- Created 2 helper functions (add/remove from index)
- Updated handle_push() to add queue to index
- Updated handle_delete() to remove from index
- Rewrote handle_list() to return real data

**Testing Results:**
✅ Push to new queue → appears in list
✅ Delete queue → removed from list (eventual consistency)
✅ Multiple queues → all listed with stats
✅ CLI list command working
✅ API returns QueueInfo (name, length, oldest_timestamp)
✅ All 21 Worker tests still passing
✅ Deployed to production

**KV Eventual Consistency:**
There may be 1-3 second delays before index updates propagate across all edge locations. This is expected KV behavior.
